COMMERCIAL PROJECTS

Invoice Robot
Robot is build as desktop application. Robot operate with input data from Microsoft mailbox, process pdf attachments to the AI processors:
  • ChatGPT
  • DocAI
AI processors recognize data from invoices. In next steps robot validate data with customer databases (customer numbers, tax ids, reference numbers, etc.). Next step is enter invoice data into existing account software and generate reports in excel format. Reports are sent by mail to the operators.

Used technologies:
Java 8, Java 21, Sikuli, AutoIt, Junit 5
Functionality:
  • read and save mail attachments in PDF from Microsoft mailbox
  • send invoices to the AI processor, receive data set
  • validate received data with customer databases to check if there is proper customer number, invoice amounts, reference number
  • operate on customer account system to book invoice into the system
  • save PDF invoices and CMR documents in archiving network disk
  • generate report in excel, send by mail to operator
DoctorBot
Robot operate on medical software Clininet as a web plugin to the Clininet web service and automate issuing medical recipes. Operator (medical doctor or nurse) have posibillity to define recipe configuration in external web config server and to use robot its enough to just one click from the drop-down menu to select recipe type and fill most of repetetive data in issuing recipe process. It help to save few minutes during patient visit.

Robot cooperate with configuration service deployed at Ubuntu server and with Chrome frontend plugin (drop-down menu for selections and staus - error presentation).

The project was initially deployed on an Apache Tomcat server for its first deployment phase. This setup allowed for straightforward deployment and integration within a traditional application server environment.

As the project evolved and scalability, consistency, and environment isolation became more important, the deployment process was migrated to a Docker container. This transition enabled easier management of dependencies, improved portability across environments.

Project was extended by additional features like:
  • automated digital signature HL7
  • multiple diagnose selection
  • referral for medical tests
Technologies:
  • Javasa 11
  • Spring framework
  • Apache Http components
  • Gson
  • Junit Jupiter
Vindication Robot
Desktop application to automate vindication reminders process. Robot operate on customer service IFS and autimatically generate reminders to the customers who have dalays in payments. Robot operate on 3 reminders level:
  • level 1 after 7 delay days - reminder print to PDF and automatically send by mail
  • level 2 afrer 14 delay days - reminder print to PDF and automatically send by mail
  • level 3 after 21 delay days - reminder print to send by traditional mail
Robot checks over 3000 records daily and generate aprox. 50-100 reminders daily.

After reminders generation process, robot create report in excel and send to the reminder operator by mail.

Robot have bulit-in scheduler to run every day in night hours before operator shift.

  • Sikuli
  • Http components
  • Apache POI
  • Junit 5
